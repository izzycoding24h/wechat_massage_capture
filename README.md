# 微信截图取证工具

这是一个面向 Windows 桌面微信的截图保存工具。它的目标很简单：当你需要保存一段微信单聊或群聊记录时，帮你连续截图、自动向上滚动、按顺序保存图片，并生成一份运行记录。

它适合普通用户使用，不需要会写命令，也不需要理解 Python。安装后打开桌面应用，按页面里的步骤操作即可。

## 为什么做这个工具

微信聊天记录有时需要作为沟通证明、结算说明、纠纷材料或内部复盘资料保存。手动一张张截图很容易出现这些问题：

- 截图顺序混乱。
- 中间漏截一页。
- 相邻截图没有重叠，后面不好核对。
- 截图太多时，不知道保存了多少张、占了多少空间。
- 多次运行后，文件夹混在一起，找不到某一次的结果。

这个工具就是为了解决这些重复、容易出错的操作。你仍然需要自己打开微信、确认聊天窗口和日期位置，工具负责自动截图、滚动、保存和记录。

## 重要提醒

- 这个工具只截取屏幕上已经能看到的微信聊天内容。
- 它不会读取微信数据库，不会破解微信，不会绕过权限，也不会修改或删除聊天记录。
- 截图只是可读副本。请保留原始电脑、手机、微信账号、聊天记录和登录环境。
- 如果材料要用于诉讼、公证或正式调查，请先咨询律师或专业机构，确认是否需要公证或司法鉴定。
- 使用前建议手动补充保存：微信账号信息页、聊天对象或群信息页、关键联系人或群成员身份页。

## 安装方式

推荐从 GitHub Releases 下载最新 Windows 安装包：

[下载 Windows 安装包](https://github.com/izzycoding24h/wechat_massage_capture/releases/latest)

如果你拿到的是安装包，直接双击安装：

```text
WechatMessageCaptureSetup-1.5.0.exe
```

安装完成后，从开始菜单或桌面快捷方式打开：

```text
微信截图取证工具
```

普通使用者不需要安装 Python，也不需要下载依赖。

如果 Windows 弹出 SmartScreen 或“不受信任”提示，这是因为当前版本没有做代码签名。确认文件来源可信后，可以选择继续运行。

## 使用前准备

正式使用前先做这几件事：

1. 打开 Windows 桌面版微信。
2. 在微信聊天列表中找到目标单聊或群聊。
3. 双击这个单聊或群聊，让它单独弹出成一个聊天窗口。
4. 不要只停留在微信主窗口左侧聊天列表里。如果没有单独弹出窗口，程序可能无法正确识别和滚动。
5. 把聊天窗口停在要截取的最新位置，通常是当前最新消息或结束日期附近。
6. 不要先手动滚到起始日期。工具会从新消息开始，一边截图一边向上滚到更早消息。

## 应用里的四个页面

应用启动后会看到左侧导航：

- `第一步 准备`
- `第二步 测试校准`
- `第三步 正式采集`

建议每次换电脑、换显示器、换微信窗口大小时，都按这个顺序操作。

## 首页

首页会说明工具用途、三步流程和停止规则。第一次使用时建议先读一遍。

最重要的是记住两点：

- 正式采集前必须先做测试校准。
- 正式采集时主要用 `Ctrl+Alt+S` 停止。

## 第一步：准备

在这个页面填写基础信息。

### 聊天窗口名

输入微信单聊或群聊窗口标题里能识别的文字。

例如窗口标题是：

```text
项目结算沟通群
```

可以填写：

```text
项目结算
```

如果程序选错窗口，通常是因为聊天没有单独弹出，或者填写的识别文字不够准确。

### 起始日期和结束日期

- `结束日期` 默认是当天。
- `起始日期` 默认是当天往前推 3 个自然月。

注意：工具不会自动识别聊天里的日期，也不会到达起始日期后自动停止。你需要在正式采集中看到聊天内容接近起始日期时，按 `Ctrl+Alt+S` 停止。

### 总保存位置

这里选择一个总目录，例如：

```text
D:\微信截图
```

每次运行时，工具会在这个目录下自动新建一个本次运行文件夹，不会把多次截图混在一起。

## 第二步：测试校准

测试校准是这个工具里最重要的一步。不同电脑、显示器、DPI、微信窗口大小都会影响滚动幅度。如果不测试就正式采集，可能会出现截图重叠不足、内容漏掉或滚动太慢。

点击：

```text
运行测试校准
```

工具会做一次简单测试：

1. 截一张 before 图片。
2. 自动滚动一次。
3. 再截一张 after 图片。

测试完成后，你需要看 before 和 after 两张图之间是否还有重复聊天内容。

### 怎么判断测试结果

合适的结果：

- before 和 after 都不是黑屏。
- 两张图之间能看到一部分相同聊天内容。
- 重叠不要太少，否则后面不好确认是否漏页。
- 重叠也不要太多，否则采集速度会变慢、截图数量会变多。

界面里显示的“图像估算移动”只是参考。微信聊天里的空白、头像、图片、表情和固定区域可能会让估算值不准确。最终以你看到的 before/after 图片为准。

### 常用参数

普通用户优先只调整这些：

- `重叠比例`：相邻截图保留多少重复内容。建议 35% 到 50%，最低 20%。
- `固定步数`：每张截图后滚动几次。越大越快，但越容易跳过内容。
- `每步滚轮力度`：单次滚轮滚动的强度。绝对值越大，移动越快。
- `截图间隔（单位：秒）`：滚动后等多久再截图。太低可能截到未刷新画面，建议从 0.2 秒开始。

如果测试发现几乎没动，可以增加固定步数或每步滚轮力度。

如果测试发现跳太多、两张图接不上，可以降低固定步数或每步滚轮力度。

如果滚动方向反了，把每步滚轮力度改成负数再测试。

### 高级参数

高级参数默认折叠。普通用户一般不需要打开。

只有遇到这些情况才建议打开：

- 截图黑屏。
- 滚动没有生效。
- 程序提前判断重复并停止。
- 鼠标没有滚到聊天内容区域。

其中 `截图方式` 的含义：

- `auto`：自动尝试多种截图方式，推荐默认使用。
- `imagegrab`：系统截图方式，通常优先。
- `pyautogui`：通用截图方式，兼容性较好。
- `mss-full`：先截全屏再裁切，适合多屏或疑难排查。
- `mss-window`：按窗口区域截图，部分电脑可能黑屏。

如果不确定，保持 `auto`。

### 截图诊断

如果测试校准图片是黑屏，点击：

```text
截图诊断
```

它不会滚动聊天，只会用不同截图方式各截一张诊断图片。诊断完成后，应用会提示诊断图片保存位置和文件名。

如果某一种截图方式正常，可以在高级参数里固定使用那一种。

## 第三步：正式采集

测试校准确认可用后，再进入正式采集。

点击：

```text
开始正式采集
```

开始前会弹出确认提示。请确认你已经知道：

- 开始后微信会反复被置前。
- 工具会点击聊天区域并向上滚动。
- 你很难稳定切回桌面应用点击停止。
- 主要停止方式是 `Ctrl+Alt+S`。

### 怎么停止

看到聊天内容滚动到起始日期附近时，按：

```text
Ctrl+Alt+S
```

按下后不会立刻停在当前画面。程序会完成当前轮截图或滚动动作，然后写入运行记录并结束。

界面会提示“正在结束中”。如果当前动作还没完成，请等几秒。

### 磁盘空间保护

正式采集页有一个设置：

```text
至少保留磁盘空间（GB）
```

默认是：

```text
10 GB
```

如果保存位置所在磁盘剩余空间低于这个值，工具会自动停止，避免长时间截图把电脑存储占满。

已经保存的截图不会被删除，仍然保留在本次运行文件夹中。

### 采集状态

正式采集时，页面会显示：

- 当前状态。
- 已保存截图数量和累计大小，例如 `5 张，共计 23.40 MB`。
- 本次运行文件夹位置。
- 运行日志。

截图大小不是固定的。聊天里如果有图片、表情或窗口尺寸变化，单张截图大小会不同。

## 输出结果在哪里

你选择的是“总保存位置”。每次运行会自动新建一个文件夹。

常见文件夹名称：

```text
正式采集_20260610_234612
测试校准_20260610_233700
截图诊断_20260610_224500
```

普通用户最常看的内容：

```text
截图\
测试校准图片\
诊断图片\
运行记录\
```

### 截图

正式采集的聊天截图保存在：

```text
截图\000001.png
截图\000002.png
截图\000003.png
```

编号越小，越靠近正式采集开始时的位置。

### 测试校准图片

测试校准会生成：

```text
测试校准图片\before.png
测试校准图片\after.png
```

用它们判断滚动幅度和重叠是否合适。

### 诊断图片

截图诊断会生成类似：

```text
诊断图片\诊断图片_20260610_2130_imagegrab.png
诊断图片\诊断图片_20260610_2130_pyautogui.png
诊断图片\诊断图片_20260610_2130_mss-full.png
诊断图片\诊断图片_20260610_2130_mss-window.png
```

这些图片用于排查黑屏或截图方式问题。

### 运行记录

`运行记录` 是给后续核对和排查用的。普通用户一般不用打开，但建议保留。

里面包括：

- `manifest.csv`：每张截图的时间、文件名、窗口位置、哈希值、是否疑似重复。
- `run.json`：本次运行参数、电脑信息、停止原因、磁盘空间记录。
- `capture.log`：运行日志。
- `sha256sums.txt`：本次运行文件夹内文件的 SHA256 清单。
- `重复截图\`：程序判断为重复或几乎未变化的截图尝试。

## 使用技巧

### 每次换环境都先测试

只要换了电脑、显示器、缩放比例、微信窗口大小，就重新跑一次测试校准。

### 微信窗口要保持可见

截图依赖屏幕上真实显示的窗口。微信不能最小化，也不要被其他窗口遮挡。

### 尽量使用主显示器

如果多屏截图异常，先把微信移动到主显示器，并让窗口最大化后再测试。

### 不要提前滚到起始日期

正式采集应该从最新消息开始，让工具往更早的消息滚动。你只需要在接近起始日期时停止。

### 先用非敏感聊天试跑

第一次使用建议找一个不敏感的聊天窗口，跑一次测试校准和少量正式采集，确认输出效果后再处理重要聊天。

### 重要材料单独备份

正式采集完成后，建议把本次运行文件夹复制一份到移动硬盘或其他位置，避免误删。

## 常见问题

### 找不到或选错微信窗口

先确认目标聊天已经从微信聊天列表中双击弹出成独立窗口。然后在 `聊天窗口名` 里填写更准确的识别文字。

### 测试校准几乎没滚动

可以增加：

- `固定步数`
- `每步滚轮力度`

也可以在高级参数里调整滚动位置比例，让鼠标落在聊天内容区域。

### 测试校准滚动太多

可以降低：

- `固定步数`
- `每步滚轮力度`

也可以提高 `重叠比例`。

### 截图是黑屏

先运行 `截图诊断`。如果所有诊断图片都是黑屏，可以尝试：

1. 确认微信窗口没有最小化。
2. 确认微信没有被其他窗口遮挡。
3. 把微信移到主显示器。
4. 保持 Windows 桌面解锁且可见。
5. 关闭微信硬件加速并重启微信。
6. 尝试用管理员身份运行应用。

### 按 Ctrl+Alt+S 后没有马上停

这是正常的。工具会完成当前轮截图或滚动动作，再写入运行记录后结束。

### 磁盘空间不足自动停止

这是保护机制。已经保存的截图仍在本次运行文件夹里。你可以换一个空间更大的保存位置后重新运行。

## 这个工具不支持什么

当前版本不支持这些能力：

- 不支持读取微信数据库。
- 不支持破解、解密或恢复已删除聊天记录。
- 不支持自动登录微信。
- 不支持自动打开某个聊天窗口。
- 不支持自动识别聊天日期并在起始日期精确停止。
- 不支持 OCR 识别图片里的文字。
- 不支持自动判断哪些截图能作为法律证据。
- 不支持自动上传云端。
- 不支持 macOS 微信。
- 不支持手机微信。

这个工具解决的是“把已经能看到的 Windows 桌面微信聊天内容稳定截图并整理保存”。它不替代律师、公证或司法鉴定流程。

## 给安装包制作者

如果你是维护者，需要重新生成安装包：

1. 在 Windows 上打开项目目录。
2. 运行 `build_windows.ps1` 生成 `dist\WechatMessageCapture\WechatMessageCapture.exe`。
3. 用 Inno Setup 打开 `installer\WechatMessageCapture.iss`。
4. 点击 `Build -> Compile`。
5. 生成的安装包在 `installer-output\` 目录下。

普通用户只需要安装包，不需要源码目录。
